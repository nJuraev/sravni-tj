# CLAUDE.md

Гайд для агентов, работающих над проектом **Sravni.tj** — агрегатор банковских продуктов (кредиты и депозиты) Таджикистана, аналог sravni.ru.

> Подробности — в [ТЗ.md](ТЗ.md) и [PRD.md](PRD.md). Этот файл — краткая справка по стеку и структуре.
>
> **Статус:** в разработке. Backend (Laravel), frontend (Vue) и парсер (Go) уже написаны. Есть админ-панель (`/admin`).

---

## Стек

| Компонент | Технология | Назначение |
|---|---|---|
| Парсер | **Go** | Сбор продуктов с сайтов банков по крону |
| Бэкенд | **PHP / Laravel** | REST API, приём и доставка заявок (email) |
| Фронтенд | **Vue.js** + Pinia + Vue I18n | Витрина сравнения, фильтры, калькулятор, формы заявок |
| БД | **PostgreSQL** | Общее хранилище: продукты, банки, заявки |
| AI для парсинга | Gemini / Qwen (бесплатный tier) | Извлечение структурированных данных из Markdown |
| Скрейпер | Firecrawl / Jina | HTML страницы банка → Markdown |
| Инфраструктура | docker-compose | Локальный запуск всех сервисов |

**Поток данных:** парсер (Go) пишет напрямую в PostgreSQL → Laravel читает из той же БД и отдаёт через REST API → Vue рендерит витрину.

## Структура (монорепо)

```
sravni-tj/
├── parser/            # Go — парсер продуктов (cron)
├── backend/           # PHP / Laravel — REST API + приём заявок + admin API (/api/admin)
├── frontend/          # Vue.js — витрина сравнения + формы заявок + админка (/admin)
└── docker-compose.yml
```

Админка живёт внутри backend/frontend (отдельных директорий нет): API под `/api/admin/*`, UI под маршрутом `/admin`.

### Ключевые таблицы PostgreSQL

- **`banks`** — банки: `name_ru`, `name_tg`, `email`, `status` (active/inactive), `is_partner`.
- **`bank_source_urls`** — URL для парсинга, привязанные к банку и категории (`credit`/`deposit`); один банк может иметь несколько URL (мультидоменность). Управляются прямым UPDATE в БД без деплоя.
- **`products`** — продукты: `category`, мультиязычные `name_ru`/`name_tg`, `status` (active/draft/hidden/outdated), `currency` (TJS/USD/EUR), диапазоны ставки/суммы/срока, `features` (jsonb).
- **`leads`** — заявки: `full_name`, `phone`, `consent` (обязателен `true`), привязка к продукту и банку.
- **`parser_runs`** — метаданные парсинга (только при флаге `PARSER_DEBUG_LOG=true`).
- **`admin_users`** — пользователи админ-панели: `name`, `email`, `password`, `role` (admin/editor), `is_active`, `api_token`.

## Ключевые правила и инварианты

- **Мультиязычность:** интерфейс и данные на двух языках — `ru` и `tg`. Поля хранятся парами `*_ru` / `*_tg`.
- **Мультивалютность:** TJS, USD, EUR. Ставка зависит от срока, суммы и валюты — это многоуровневые тарифные сетки, не одно число.
- **Заявки:** `consent = true` обязателен, иначе `POST /api/leads` → 422. Заявка всегда уходит на email банка (`banks.email`), независимо от `is_partner`.
- **Парсер:** AI вызывается в режиме structured output; числовые поля валидируются на диапазон (ставка 0–100%, сумма > 0) до записи в БД (анти-галлюцинации).
- **API отдаёт** только продукты со `status = active` и банком `status = active`.
- **Калькулятор** считается полностью на клиенте (аннуитет для кредита, простые/сложные проценты для депозита) — не парсится.
- **Без авторизации публичных пользователей** на витрине (заявки анонимны). Управление данными — через **админ-панель** (`/admin`) либо прямой доступ к БД.
- **Админка:** аутентификация через таблицу `admin_users` (встроенный token-guard Laravel, guard `admin`, токен в `Authorization: Bearer`). Роль `admin` управляет пользователями; `editor` — без раздела пользователей. CRUD по банкам/продуктам пишет в guarded-модели через `forceFill`.

## Запреты

- **Не читать и не изменять `.env`-файлы** (и любые файлы с секретами/ключами) — ни в одном из сервисов.
